WebAppGPX — État complet de l'application
Stack technique
- React 19 + TypeScript 6.0 (strict : verbatimModuleSyntax, erasableSyntaxOnly, noUnusedLocals, noUnusedParameters)
- Vite 8 avec @cloudflare/vite-plugin v1.36.3 pour le dev
- Cloudflare Pages (SPA, not_found_handling: single-page-application, nodejs_compat)
- Tailwind v3 (PostCSS), pas de librairie UI externe
- Pas de test runner, pas de Zustand — useState + hooks personnalisés

Authentification (AuthContext.tsx)
- Email/password via supabase.auth.signInWithPassword / signUp
- Session stockée en RAM (memoryStorage), perdue au F5
- autoRefreshToken: true — géré par Supabase
- Cycle d'init :
1. supabase.auth.getUser() → vérifie cookie auth
2. onAuthStateChange → INITIAL_SESSION + SIGNED_IN (restauration via cookie)
3. loadProfile() → ensureProfile() (upsert dans profiles)
- RLS : is_admin() SECURITY DEFINER sur toutes les fonctions
Fonctionnalités
Waypoints : drag-and-drop (@dnd-kit), ajout/suppression, numérotation cercles colorés
Groupes (routeStore.ts) :
- Création : Shift+drag d'un waypoint sur un autre dans la sidebar
- Opérations : extend, split, dissolve, reorder, fold/unfold
- Métriques : vitesse personnalisée, durée, mergeOverlappingGroups, rebaseGroupIndices
- Planning : lecture depuis les groupes (pas localStorage)
Sauvegarde en ligne (lib/savedRoutes.ts) :
- Table saved_routes (JSONB waypoints/groups/segment arrays)
- Auto-save BROUILLON (debounce 2s) → pas de binding savedRouteId au chargement
- Conflit de nom géré : vérification + auto-rename "(2)", "(3)" sur refus
- CRUD via SavedRoutesModal (recherche, boutons 💾/📂)
Lieux enregistrés (services/savedPlaces.ts) :
- Table saved_places avec RLS (SELECT tous, INSERT/UPDATE/DELETE par owner)
- Toggle Mes/Tous lieux, Edit/Delete masqué pour les autres
Géocodage (services/geocoding.ts) :
- Proxy Nominatim via Pages Function (prod) + proxy Vite (dev)
- Rate limit 1.1s entre appels
- Cache 3 niveaux : mémoire → saved_places → locality_cache Supabase
Problème résolu : Background tab throttling
- Cause : le timer interne autoRefreshToken de Supabase est throttlé par Chrome en arrière-plan → _refreshTokenPromise ne résout jamais → toutes les requêtes Supabase pendent
- Solution : au retour d'onglet (visibilitychange), recreateSupabaseClient() crée une nouvelle instance Supabase (état interne vierge). Le userRef filtre les SIGNED_IN redondants émis par l'ancien client → pas de cascade React (unmount), les données de la route sont préservées
- Mécanisme clé : export let supabase (live binding ES module) — tous les imports reçoivent automatiquement la nouvelle instance
